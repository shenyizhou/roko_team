# 队伍角色与体系识别系统 设计文档

**日期**: 2026-05-01
**版本**: v1.0
**状态**: 已批准

## 1. 背景与目标

目前的队伍推荐系统主要基于：
- 精灵个体评分
- 属性联防
- 能量协同
- 负面特性惩罚

但缺少从实际对战构筑思路的评估维度：
1. **角色配置**：队伍是否有合格的清场手、首发、辅助、扫尾
2. **体系识别**：队伍是否构成完整的战术体系（如毒队、雷暴队等）
3. **首领化冲突**：是否有多个依赖首领化的精灵互斥

本设计目标是将这些实战构筑思路量化，整合到现有评分系统中。

## 2. 整体架构

### 2.1 模块划分

```
models/
  ├── team_analyzer.py      # 现有，扩展整合新功能
  ├── pet_scorer.py         # 现有，扩展角色标签计算
  ├── role_classifier.py    # 新增：精灵角色分类器
  └── system_detector.py    # 新增：队伍体系识别与评分器
```

### 2.2 数据流

1. `role_classifier.py` 为每只精灵计算4个角色维度得分，输出1-2个主角色标签
2. `system_detector.py` 识别队伍所属的体系，计算体系完成度，检测首领化冲突
3. `team_analyzer.py` 整合以上结果，输出完整的队伍分析报告

### 2.3 评分公式

新的队伍综合评分公式：
```
队伍总分 = 原评分 × 0.6
          + 角色配置合理性评分 × 0.2
          + 体系完成度评分 × 0.2
```

- 原有个体质量和联防占主要权重（60%）
- 角色配置和体系各占20%，确保能显著影响最终排名

## 3. 精灵角色分类器 (role_classifier.py)

### 3.1 角色定义与判定标准

为每只精灵计算4个维度的得分（0-100），取最高的1-2个作为角色标签。

#### 清场手 (Cleaner)

按优先级从高到低检测，满足条件加分：

| 类型 | 判定条件 | 分值 |
|------|---------|------|
| 强化超速型 | 有加速技能（速度+50%等）+ 强化后速度 > 130 + 高威力输出 | 80-100 |
| 高速强化型 | 基础速度 ≥ 120 + 有强化技能（物攻/魔攻+） | 60-80 |
| 特性优秀型 | 特性含「能量」「力竭」「斩杀」等特殊机制关键词 | 50-70 |
| 肉盾强化型 | HP×防御 ≥ 25000 + 有回复技能 + 有强化技能 | 40-60 |

#### 首发/开路 (Starter)

| 判定条件 | 分值 |
|---------|------|
| 有天气设置技能（「将天气改为」） | 30 |
| 有迅捷技能（0费先出手） | 30 |
| 有控制技能（眩晕、寄生等） | 20 |
| 速度 ≥ 110 | 20 |

#### 辅助 (Support)

| 判定条件 | 分值 |
|---------|------|
| 有驱散印记/清增益/清减益技能 | 30 |
| 有给队友增益的技能（力量增效等） | 25 |
| 特性与其他精灵有联动（如印记转化） | 25 |
| 高双防低攻（身板硬输出弱） | 20 |

#### 扫尾 (Finisher)

| 判定条件 | 分值 |
|---------|------|
| 有斩杀类技能（敌方HP越低伤害越高） | 40 |
| 有先制技能（迅捷、优先出手） | 30 |
| 高速高攻但身板脆 | 30 |

### 3.2 输出格式

```python
{
    "role_scores": {
        "cleaner": 85,
        "starter": 70,
        "support": 30,
        "finisher": 45
    },
    "primary_roles": ["cleaner", "starter"],  # 取前2名
    "role_details": "强化超速型清场，有天气首发能力"
}
```

## 4. 体系识别与评分器 (system_detector.py)

### 4.1 体系清单与规则

每个体系定义核心精灵（必须有）和协同精灵（满足加分）。

| 体系名称 | 核心精灵（必须有） | 协同规则 | 完成度计算 |
|---------|-------------------|---------|-----------|
| **毒队** | 厉毒修萝 + 裘卡 | 影狸、其他毒系精灵、有中毒相关技能 | 核心40% + 协同30% + 印记驱散30% |
| **星陨队** | 怖哭菇 + 落陨星兔 | 帕尔、暮星辰、圣羽翼王、有星陨技能 | 核心50% + 炸能量25% + 迅捷触发25% |
| **独角兽队** | 白金独角兽 | 有折射技能、其他独角兽、有清增益 | 首领化核心60% + 协同40% |
| **恶魔狼王队** | 恶魔狼 + 奇瓦亚 | 音速犬（首领候补）、送死触发特性 | 核心50% + 送死组件30% + 多首领候补20% |
| **三三队** | 梦想三三 | 有甜心技能、有续航、有强化 | 首领化核心60% + 续航协同40% |
| **火神队** | 火神 | 有火天气、有火系队友 | 首领化核心50% + 火系协同50% |
| **地武队** | 棋绮后 | 有地刺技能、有地/武系输出 | 首领化核心40% + 每只地刺精灵+15% |
| **圣羽翼王队** | 圣羽翼王 | 岚鸟、其他翼系精灵、有迅捷技能 | 核心50% + 每只翼系协同+25% |
| **圣剑队** | 圣剑 | 有休息回复、有强化 | 首领化核心70% + 生存能力30% |
| **雷暴队** | 闪电鳗鱼 + 星光狮 | 有雷暴天气、有电系迸发 | 核心60% + 天气协同40% |

### 4.2 首领化冲突检测

- 统计队伍中「依赖首领化」的精灵数量
- 超过1个 → 警告：「队伍中有X个依赖首领化的精灵，互斥！建议只留一个核心」
- **例外**：恶魔狼王队允许带首领候补，不算冲突

### 4.3 体系评分逻辑

1. 对所有10个体系分别计算完成度百分比
2. 输出排名前3的体系给用户
3. 完成度 ≥ 70% 判定为「完整体系」，享受体系加成
4. 队伍体系分 = 最高体系完成度

## 5. 队伍分析报告输出变化

在原有报告基础上，新增三个板块：

### 5.1 角色配置分析

```
【角色配置分析】
  清场手: 闪电鳗鱼、黑羽夫人 (2只) ✓
  首发: 闪电鳗鱼、星光狮 (2只) ✓
  辅助: 影狸、寂灭骨龙 (2只) ✓
  扫尾: 贝古斯 (1只)
  配置合理性: 85/100 - 角色均衡，梯队完整
```

### 5.2 体系识别结果

```
【体系识别结果】
  检测到体系:
    1. 雷暴队 - 完成度 85% (完整体系 ✓)
       核心: ✓ 闪电鳗鱼 ✓ 星光狮
       协同: ✓ 雷暴天气 × 电系迸发
    2. 毒队 - 完成度 30% (不完整)
       核心: × 厉毒修萝 ✓ 影狸

  建议: 去掉贝古斯，换成厉毒修萝，可组成完整毒+雷暴双核体系
```

### 5.3 首领化冲突检测

```
【首领化分析】
  依赖首领化精灵: 恶魔狼、梦想三三、火神 (3只) ⚠
  警告: 首领化只能用一次，建议只保留1个核心首领化精灵
  推荐: 留恶魔狼为核心，其他替换
```
