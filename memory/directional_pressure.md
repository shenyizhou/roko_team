# 定向施压战术体系

## 核心概念
### 起点 vs 终点
- **起点**：率先发起对check/counter攻击的精灵
- **终点**：最终突破并收益的精灵（一般是首领）
- **建队思路**：围绕首领（终点）组建进攻冗余的起点们

### 联攻轴定义
两只或多只精灵**共享同一个counter精灵**，它们互为进攻冗余。

#### 例子1：恶魔狼王 + 音速犬
- **终点**：恶魔狼王，配招 = 恶系 + 火系 + 普通系
- **counter**：绒光优优（光+地，能挡恶/火/普通）
- **起点**：音速犬，**也被绒光优优counter**
- **结论**：音速犬就是恶魔狼王的进攻冗余 → 它们构成**联攻轴**
- **实战**：音速犬先上场逼出绒光优优，压低血量 → 恶魔狼王登场突破

#### 例子2：电球咩咩 + 海豹船长
- **共同counter**：寂灭骨龙、幻影灵菇（能同时挡电、武、水系）
- **结论**：它们互为进攻冗余，构成联攻轴

### 多联攻轴补齐
如果一个联攻轴的两只精灵，对手有两个不同counter导致联攻轴失效 → 继续寻找新的联攻轴，用**三只精灵面对两只counter**。

## 建队风格分类

### Hyper Offense (HO) - 极致进攻队
- **定义**：队伍里4-6只都是核心进攻精灵
- **核心要求**：每1只进攻精灵都至少有2只以上队友存在进攻冗余
- **战术目标**：实现全方位的定向施压战术
- **打法特点**：
  - 基本不换人联防
  - 劣势对位直接卖掉自己去换血
  - 极致的进攻，靠压迫感取胜

### Offense - 攻队
- **定义**：带了2只以上联防轴，不再全是进攻位
- **特点**：攻守平衡，有一定的防守兜底能力

## 算法评分应用

### 1. 联攻轴检测（核心）
对于每只精灵A，找出所有能和A共享counter的精灵B → 它们构成联攻轴。
- 联攻轴对数越多 → 队伍进攻协同性越好

### 2. HO队伍检测与加分
- 4只以上核心进攻精灵 + 每只有2+冗余队友 → 大幅加分
- HO队伍应该减少联防面的评分权重，大幅增加进攻冗余的评分权重

### 3. 起点/终点配合加分
- 围绕首领精灵（终点）有3+个起点精灵 → 大幅加分
- 起点/终点共享counter的数量越多 → 联攻强度越高 → 加分越高

### 4. 联攻轴补齐检测
3只精灵面对2只共同counter → 形成完整的定向施压链 → 额外加分
