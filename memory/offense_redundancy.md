# 进攻冗余（Offense Redundancy）概念

## 核心思想
联防 vs 突破的平衡：
- **纯联防思路**：各种属性的精灵都有 → 更好地联防对手的进攻
- **问题**：缺乏手段突破对手的防守 → 进攻端乏力

## 进攻冗余定义
不要只去找"counter对方counter位的精灵"，而是：
1. 在我们的清场手登场之前
2. 使用**类似属性的输出技能**逼迫对手的counter位精灵登场
3. 通过**多只精灵同时对一只防守精灵施压**
4. 从而撕开对手的防线

## 实战应用
### 例子：围绕圣羽翼王构建进攻冗余
- **核心清场手**：圣羽翼王（翼系输出）
- **counter位**：冰系/电系/机械系等能挡翼系的精灵
- **进攻冗余构建**：队伍中再加入2-3只也带翼系输出技能的精灵（如黑羽夫人、岚鸟、噼啪鸟等）
- **效果**：多只精灵同时用翼系技能打击对方的counter位 → 迫使对方counter位血量下降 → 圣羽翼王登场时可以直接推队

### 例子：围绕地系核心构建进攻冗余
- **核心清场手**：波多西/嗜波螺（地系输出）
- **counter位**：草系/虫系/翼系等能挡地系的精灵
- **进攻冗余构建**：多只精灵都带地刺/鸣沙陷阱等地系技能

## 组队算法应用
1. **同属性输出集中度指标**：同一属性的输出技能在队伍中出现的次数
   - 3次及以上：良好的进攻冗余 → 加分
   - 5次及以上：极致进攻冗余 → 大幅加分（纯属性队）

2. **核心清场手的同属性支援**：
   - 核心是翼系 → 队伍中其他精灵也尽量带翼系输出技能
   - 核心是地系 → 队伍中其他精灵也尽量带地系输出技能

3. **属性权重调整**：
   - 不是属性覆盖越广越好
   - 集中1-2个核心属性进行"饱和进攻"更有实战价值

4. **平衡取舍**：
   - 纯属性队（6只同核心属性）：进攻拉满，但联防可能有漏洞
   - 双属性队（4+2）：平衡型，既有进攻冗余又有一定联防

## 三大进攻冗余战术

### 战术一：定向施压（联攻轴）
- **核心机制**：起点精灵和终点精灵**共享同一个counter**
- **特点**：起点和终点可以互换，根据场上形势动态变化
- **实现方式**：
  - 相同属性精灵（最简单思路）
  - 不同属性但共享counter的精灵
- **进阶**：多联攻轴补齐（3只精灵面对2只共同counter）

### 战术二：控制战术（让counter失能）
- **核心机制**：通过控制能力让counter位失去应对能力，从而让终点安全登场
- **关键手段**：
  - **无法下场消除的状态**：萌化、冰冻、隐藏条款
  - **阻止恢复能量**：幽影树的勾魂、幻影灵菇的惊吓盒子
  - **打断聚能**：地刺等技能
- **与定向施压的区别**：
  - 定向施压：起点/终点可动态互换
  - 控制战术：起点的一切行动都是为了终点铺路，单向关系

### 战术三：地雷战术（信息差）
- **核心机制**：携带原本不太会携带的技能，利用信息差打出克制counter的效果
- **实现方式**：
  - 用愿力强化
  - 携带克制counter的血脉
- **限制**：愿力和首领冲突，只能在没有首领化的队伍里使用
- **与控制战术的共同点**：都是单向地为终点铺路
